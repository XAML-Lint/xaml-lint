# LX800: Uno platform XML namespace must be mc:Ignorable

<!-- Upstream: RXT700. -->

## Cause

A root element declares an `xmlns:` prefix pointing at a Uno Platform
platform-specific URI, but the prefix is not listed in the root's
`mc:Ignorable` attribute.

## Rule description

Uno Platform ships a set of XAML namespace URIs that target individual
platforms — `http://uno.ui/android`, `.../ios`, `.../wasm`,
`.../macos`, `.../skia`. Elements and attributes in these namespaces
are only understood by the Uno XAML parser; stock WinUI/UWP/WPF parsers
will fail to load the document unless the prefix is declared ignorable.

The fix is to add every Uno-platform prefix to the root element's
`mc:Ignorable` list.

**Covered Uno URIs (as of 2026-04):**

| URI | Platform |
|---|---|
| `http://uno.ui/android` | Android |
| `http://uno.ui/ios` | iOS |
| `http://uno.ui/wasm` | WebAssembly |
| `http://uno.ui/macos` | macOS |
| `http://uno.ui/skia` | Skia |

New Uno platforms added after this release won't be flagged until the
allowlist is updated.

```xaml
<!-- non-compliant: xmlns:android declared but not ignorable -->
<Page xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
      xmlns:android="http://uno.ui/android">
</Page>

<!-- compliant -->
<Page xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
      xmlns:android="http://uno.ui/android"
      xmlns:ios="http://uno.ui/ios"
      xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006"
      mc:Ignorable="android ios">
</Page>
```

## How to fix violations

1. Declare the markup-compatibility namespace if you haven't:
   `xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006"`.
2. Add every Uno-platform prefix you use to `mc:Ignorable` on the root,
   space-separated.

## How to suppress violations

For a single document:

```xaml
<!-- xaml-lint disable LX800 -->
<Page xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
      xmlns:android="http://uno.ui/android">
</Page>
```

For a whole project:

```json
{ "rules": { "LX800": "off" } }
```
