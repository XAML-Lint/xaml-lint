# LX103: Grid.ColumnSpan exceeds available columns

<!-- Upstream: RXT104. -->

## Cause

A child element of a `<Grid>` sets `Grid.ColumnSpan="N"` where `N` is greater than the
total number of columns the Grid declares. A Grid with no `<Grid.ColumnDefinitions>` (or
no `ColumnDefinitions="..."` shorthand) has one implicit column, so `Grid.ColumnSpan="2"`
exceeds it.

## Rule description

`Grid.ColumnSpan` is an attached integer property that specifies how many columns a child
element occupies horizontally. A span equal to the total column count is legal — the
common "row header spanning all columns" idiom. A span greater than the total column count
almost always indicates the author intended more columns or miscounted.

```xaml
<Grid xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation">
    <Grid.ColumnDefinitions>
        <ColumnDefinition />
        <ColumnDefinition />
    </Grid.ColumnDefinitions>
    <!-- non-compliant: Grid.ColumnSpan="3" but only 2 columns exist -->
    <Rectangle Grid.ColumnSpan="3" Fill="LightGray" />
</Grid>
```

```xaml
<Grid xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation">
    <Grid.ColumnDefinitions>
        <ColumnDefinition />
        <ColumnDefinition />
    </Grid.ColumnDefinitions>
    <!-- compliant: Grid.ColumnSpan="2" matches the column count; spans the whole Grid -->
    <Rectangle Grid.ColumnSpan="2" Fill="LightGray" />
</Grid>
```

The check considers the span value in isolation (`span > columnCount`) and does not combine
`Grid.Column` with `Grid.ColumnSpan`. The combined overflow case (a non-zero `Grid.Column`
that shifts the span past the end) is a planned future check; today, [LX101](LX101.md)
separately flags a starting column that is itself out of range, independent of any span.
Both attribute and element syntaxes are supported, and the WinUI/UWP
`ColumnDefinitions="*,Auto"` shorthand is honored.

## How to fix violations

Either add more `<ColumnDefinition>` entries until the span fits, or reduce the
`Grid.ColumnSpan` value to the actual column count.

## How to suppress violations

For a single element (the next `XElement` after the comment):

```xaml
<!-- xaml-lint disable once LX103 -->
<Rectangle Grid.ColumnSpan="99" />
```

For a block of markup:

```xaml
<!-- xaml-lint disable LX103 -->
<Rectangle Grid.ColumnSpan="99" />
<Border Grid.ColumnSpan="99" />
<!-- xaml-lint restore LX103 -->
```

For a whole file or project, edit `xaml-lint.config.json`:

```json
{ "rules": { "LX103": "off" } }
```
