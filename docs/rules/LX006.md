# LX006: Internal error in rule

<!-- Emitted by the RuleDispatcher's try/catch wrapper. -->

**Applies to:** `Avalonia`  `MAUI`  `Uno`  `UWP`  `WinUI 3`  `WPF`

## Cause

A rule implementation threw an unhandled exception while analyzing a file. This is always a bug in the rule, not in the user's XAML.

## Rule description

Every rule invocation is wrapped in a `try/catch`. If the rule throws, the exception type and short message are captured into an `LX006` diagnostic at the start of the file, the offending rule is skipped for that file, and linting continues on the next rule and the next file. Diagnostic output stays clean — the crash doesn't leak a stack trace to the user's output.

Run with `--verbosity diagnostic` to see the full stack trace on stderr.

## How to fix violations

File a bug report at <https://github.com/XAML-Lint/xaml-lint/issues> with:

- The `LX006` message (includes the exception type and rule ID).
- The XAML snippet that triggered it (minimal repro preferred).
- Output of `xaml-lint --version` and `dotnet --info`.

## How to suppress violations

```json
{ "rules": { "LX006": "off" } }
```

Silencing `LX006` lets crashes pass CI silently — only do this if you absolutely know what you're doing and have an open tracking issue.
