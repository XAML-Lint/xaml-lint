# LX003: Malformed configuration

<!-- Emitted by the ConfigLoader. -->

**Applies to:** `Avalonia`  `MAUI`  `Uno`  `UWP`  `WinUI 3`  `WPF`

## Cause

`xaml-lint.config.json` is unreadable, contains malformed JSON, names an unknown `defaultDialect`, has a malformed glob in `overrides[].files`, or references a rule ID in `rules` that the catalog doesn't recognize.

## Rule description

At `error` severity (the default for parse/schema failures) this aborts linting with exit code 2 — no files are processed because the effective rule severities can't be determined. At `warning` severity (for unknown rule IDs, which may be forward-compat entries for an upgraded tool) linting continues and the unknown entry is ignored.

```json
// non-compliant: unknown severity string
{ "rules": { "LX001": "critical" } }
```

```json
// compliant
{ "rules": { "LX001": "error" } }
```

## How to fix violations

Read the emitted message. It includes the config file path and the failing key or line. Common fixes:

- Replace an invalid severity string with `off`, `info`, `warning`, or `error`.
- Replace an invalid dialect with one of `wpf | winui3 | uwp | maui | avalonia | uno`.
- Run the config through `python -m json.tool` or `jq .` to catch JSON syntax errors.

## How to suppress violations

Not suppressible — a broken config is a build-time failure by design.
