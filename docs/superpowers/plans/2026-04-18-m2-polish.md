## Task 3: LX400 — simplify vestigial namespace filter and test the skip

**Bug:** The namespace filter in `LX400_HardcodedString.Analyze` reads:

```csharp
if (!attr.Name.NamespaceName.Equals(string.Empty, StringComparison.Ordinal) &&
    !attr.Name.NamespaceName.Equals(XamlNamespaces.WpfPresentation, StringComparison.Ordinal))
    continue;  // attached properties and other namespaces aren't the target
```

The `WpfPresentation` branch is dead code: unprefixed attributes on a WPF-default-namespace element have `NamespaceName == ""` per XML rules, not the element's default namespace. The only way to reach an attribute whose `NamespaceName` is `WpfPresentation` is a contrived `xmlns:wpf="…"` + `wpf:Text="…"` — essentially never written. The comment also misdescribes the check ("attached properties" are namespaced, but so is anything with a prefix).

## Task 4: Meta-test — category column in AnalyzerReleases matches `XamlLintCategory.ForId`

**Gap:** A row like `LX300 | Tool | …` would slip through M1/M2 reviews — no test correlates the category written in `AnalyzerReleases.Shipped.md`/`Unshipped.md` with `XamlLintCategory.ForId(id)`. The design spec says category derives from the hundreds digit; enforce that invariant on the release files too.

**Files:**
- Modify: `tests/XamlLint.Core.Tests/Meta/CatalogMetaTest.cs`

- [ ] **Step 1: Add the meta-test**

Append this method inside the `CatalogMetaTest` class (same file, same class). Keep the existing `FindRepoRoot` helper.

```csharp
[Fact]
public void Analyzer_release_category_column_matches_category_derivation()
{
    var repoRoot = FindRepoRoot();
    var shipped = File.ReadAllText(Path.Combine(repoRoot, "AnalyzerReleases.Shipped.md"));
    var unshipped = File.ReadAllText(Path.Combine(repoRoot, "AnalyzerReleases.Unshipped.md"));
    var combined = shipped + "\n" + unshipped;

    // Match rows like:   LX300   | Naming   | Warning  | Description
    // The first two pipe-delimited cells are the ID and the category.
    var row = new System.Text.RegularExpressions.Regex(
        @"^(?<id>LX\d{3})\s*\|\s*(?<category>\w+)\s*\|",
        System.Text.RegularExpressions.RegexOptions.Multiline);

    foreach (System.Text.RegularExpressions.Match m in row.Matches(combined))
    {
        var id = m.Groups["id"].Value;
        var writtenCategory = m.Groups["category"].Value;
        var expected = XamlLintCategoryNames.NameOf(XamlLintCategoryExtensions.ForId(id));
        writtenCategory.Should().Be(expected,
            $"AnalyzerReleases row for {id} must list category '{expected}', got '{writtenCategory}'");
    }
}
```

```csharp
[Fact]
public void Every_rule_appears_in_its_category_overview_page()
{
    var repoRoot = FindRepoRoot();
    foreach (var m in Rules)
    {
        var category = XamlLintCategoryExtensions.ForId(m.Id);
        var overviewFile = Path.Combine(
            repoRoot, "docs", "rules",
            XamlLintCategoryNames.NameOf(category).ToLowerInvariant() + ".md");

        File.Exists(overviewFile).Should().BeTrue(
            $"category overview page '{overviewFile}' must exist for rule {m.Id}");

        var text = File.ReadAllText(overviewFile);
        text.Should().Contain($"[{m.Id}]",
            $"{overviewFile} must link to rule {m.Id} (expected a '[{m.Id}](…)' markdown link)");
    }
}
```
