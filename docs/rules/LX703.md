# LX703: Entry lacks accessibility description

## Cause

A MAUI `<Entry>` has no attribute that describes it for assistive
technologies.

## Rule description

Screen readers on Android (TalkBack) and iOS (VoiceOver) need a label
or description to announce a text input's purpose. A bare `<Entry>`
is opaque — the user hears only "text field".

**The rule is off by default in `:recommended`** because most codebases
have not done a deliberate accessibility pass; enabling it there would
flood output. Teams who want to enforce a11y should extend `:strict`, or
explicitly set `LX703: warning` in their config.

**Suppressors:**

- `SemanticProperties.Description` — MAUI's preferred short AT name.
  Any non-empty value suppresses.
- `SemanticProperties.Hint` — longer AT description / hint.
  Any non-empty value suppresses.
- `AutomationProperties.Name` — cross-platform AT name;
  MAUI supports it for interop. Any non-empty value suppresses.
- `Name` or `x:Name` — the runtime name is what most MAUI platforms
  fall back to when no semantic property is set.

```xaml
<ContentPage xmlns="http://schemas.microsoft.com/dotnet/2021/maui">
    <!-- non-compliant -->
    <Entry />

    <!-- compliant: short AT name -->
    <Entry SemanticProperties.Description="Username" />

    <!-- compliant: name + hint -->
    <Entry SemanticProperties.Description="Password"
           SemanticProperties.Hint="Enter the password for your account." />
</ContentPage>
```

## How to fix violations

Add at least `SemanticProperties.Description="…"`. Use `Hint` in
addition when the field needs more context than a short label (e.g.
format requirements, placeholder explanations).

## How to suppress violations

For a single element:

```xaml
<!-- xaml-lint disable once LX703 -->
<Entry />
```

For a whole file or project:

```json
{ "rules": { "LX703": "off" } }
```
