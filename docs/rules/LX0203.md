# LX0203: x:Reference target does not exist

**Applies to:** `Avalonia`  `MAUI`  `Uno`  `UWP`  `WinUI 3`  `WPF`

## Cause

An `{x:Reference Foo}` / `{x:Reference Name=Foo}` / `{Reference Foo}` markup-extension
value names a target (`Foo`) that is not declared as an `x:Name` or `Name` on any element
in the reference site's XAML name scope. On dialects where `x:Reference` is resolved at
load time, the XAML parser throws `XamlParseException` when the name is missing. On other
dialects the assignment silently stays at its default value.

## Rule description

`x:Reference` is the XAML 2009 element-reference primitive, used as:

- `{x:Reference Foo}` — positional form, the most common spelling.
- `{x:Reference Name=Foo}` — named-argument form.
- `{Reference Foo}` — unprefixed form, valid when `x:` is the default namespace
  (MAUI XAML 2009 files using `xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"`).

All three are checked against the same XAML namescope rules used by
[LX0202](LX0202.md):

- Root scope — the document.
- Template scopes — `ControlTemplate`, `DataTemplate`, `ItemsPanelTemplate`,
  `HierarchicalDataTemplate` each open a nested scope that isolates names from the
  outer scope.

The rule also inspects element-form `<x:Reference Name="X"/>` nodes —
rare in practice, but valid where the reference is a property-element
value (`<TextBox.DataContext><x:Reference Name="..."/></TextBox.DataContext>`).
The same scope rules apply. Nested markup extensions inside an attribute
value (e.g. `{Binding Source={x:Reference Foo}}` — the dominant
cross-namescope idiom in MAUI) are also walked, so the inner reference
is checked.

```xaml
<!-- non-compliant: "Ghost" is not a named element in this scope -->
<StackPanel xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
            xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">
    <Label x:Name="Header" />
    <TextBox DataContext="{x:Reference Ghost}" />
</StackPanel>
```

```xaml
<!-- non-compliant: WPF Label.Target to a missing element -->
<Label Target="{x:Reference Missing}" Content="_User" />
```

```xaml
<!-- compliant: Header is in the same scope -->
<StackPanel xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
            xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">
    <Label x:Name="Header" />
    <TextBox DataContext="{x:Reference Header}" />
</StackPanel>
```

### What is out of scope

- **`{Binding ElementName=…}` targets.** Covered by [LX0202](LX0202.md).

## How to fix violations

1. Check the reference for typos.
2. If the target is in a different template scope, move the reference into the same
   template or switch to `{TemplateBinding …}` / `{RelativeSource …}`.
3. If the target truly lives cross-scope, you cannot use `{x:Reference}` to reach it —
   consider exposing the value via a shared `DataContext` or a dependency property the
   template can bind to.

## How to suppress violations

For a single element:

```xaml
<!-- xaml-lint disable once LX0203 -->
<TextBox DataContext="{x:Reference Ghost}" />
```

For a block:

```xaml
<!-- xaml-lint disable LX0203 -->
<TextBox DataContext="{x:Reference First}" />
<TextBox DataContext="{x:Reference Second}" />
<!-- xaml-lint restore LX0203 -->
```

For a whole file or project:

```json
{ "rules": { "LX0203": "off" } }
```
