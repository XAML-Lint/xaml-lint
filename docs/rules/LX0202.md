# LX0202: Binding ElementName target does not exist

**Applies to:** `Avalonia`  `MAUI`  `Uno`  `UWP`  `WinUI 3`  `WPF`

## Cause

A `{Binding ElementName=Foo, …}` markup-extension value names a target (`Foo`) that is not
declared as an `x:Name` or `Name` on any element in the reference site's XAML name scope.
At runtime the binding resolves to nothing: the target property stays at its default
value, and the binding engine emits a "cannot find source for binding" error into the
debug output.

## Rule description

XAML namescopes follow template boundaries:

- **Root scope** — the document.
- **Template scopes** — `ControlTemplate`, `DataTemplate`, `ItemsPanelTemplate`,
  `HierarchicalDataTemplate` each open a nested scope. Names inside are isolated from
  the outer scope, and the outer scope is not visible from inside.

The rule inspects every attribute whose value parses as `{Binding …}` with an
`ElementName=` named argument, and resolves the target name against the
reference-site element's enclosing scope. A name that doesn't resolve is flagged
regardless of where in the document the binding appears.

```xaml
<!-- non-compliant: "Ghost" is not a named element in this scope -->
<StackPanel xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
            xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">
    <Label x:Name="Header" Content="Hello" />
    <TextBox Text="{Binding ElementName=Ghost}" />
</StackPanel>
```

```xaml
<!-- non-compliant: "OuterLabel" is in the root scope but invisible inside the template -->
<Button xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">
    <Button.Template>
        <ControlTemplate>
            <TextBlock Text="{Binding ElementName=OuterLabel}" />
        </ControlTemplate>
    </Button.Template>
</Button>
```

```xaml
<!-- compliant: "Header" is declared in the same scope as the reference -->
<StackPanel xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
            xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">
    <Label x:Name="Header" Content="Hello" />
    <TextBox Text="{Binding ElementName=Header, Path=Content}" />
</StackPanel>
```

### What is out of scope

- **Nested markup extensions.** Only the outer attribute value is inspected. An
  `ElementName=` buried inside a nested converter or source expression is not checked —
  e.g. the inner reference in `{Binding Source={x:Reference Foo}}`.
- **`{x:Bind}` targets.** `x:Bind` has no `ElementName` argument; named elements are
  reached by typed path. Covered separately by the x:Bind compiler.
- **`{x:Reference}` targets.** Covered by [LX0203](LX0203.md).
- **Pure data `{Binding}` without `ElementName`.** Pure path bindings have no
  compile-time target to validate.

## How to fix violations

1. Check for typos in the `ElementName=` value — the runtime does not warn about
   close matches.
2. If the target is inside a `ControlTemplate` / `DataTemplate`, use
   `{TemplateBinding …}` or `{RelativeSource TemplatedParent}` instead of
   `ElementName=` — template scopes cannot reach the outer scope.
3. If the target is in a different template, consider `{RelativeSource AncestorType=…}`
   to walk up the logical tree.

## How to suppress violations

For a single element:

```xaml
<!-- xaml-lint disable once LX0202 -->
<TextBox Text="{Binding ElementName=Ghost}" />
```

For a block:

```xaml
<!-- xaml-lint disable LX0202 -->
<TextBox Text="{Binding ElementName=First}" />
<TextBox Text="{Binding ElementName=Second}" />
<!-- xaml-lint restore LX0202 -->
```

For a whole file or project:

```json
{ "rules": { "LX0202": "off" } }
```
