# LX004: Cannot read file

<!-- Emitted by the file enumerator. -->

## Cause

A path given to `xaml-lint` exists but cannot be read — typically permissions, a lock held by another process, or a symbolic link with a broken target.

## Rule description

The file is skipped for rule evaluation. Other files in the invocation continue linting; the exit code is raised to 2 to signal tool-level failure.

## How to fix violations

- Check file permissions (`ls -l` / `Get-Acl`).
- Close any editor or build process that may hold the file locked.
- Replace or remove broken symlinks.

## How to suppress violations

Don't — `LX004` indicates a real I/O failure. If you deliberately want to skip unreadable files, use `--exclude` to filter them out of the invocation.
