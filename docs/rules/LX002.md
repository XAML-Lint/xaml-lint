# LX002: Unrecognized pragma directive

<!-- Emitted by the PragmaParser. -->

**Applies to:** `Avalonia`  `MAUI`  `Uno`  `UWP`  `WinUI 3`  `WPF`

## Cause

A XAML comment whose body starts with `xaml-lint` did not match the pragma grammar (spec §3.4). Typical causes: typos (`disableonce` instead of `disable once`), unsupported directives, or rule IDs that don't match the `[A-Z]+\d+` pattern.

## Rule description

Silent misuse of pragmas is worse than a noisy warning: a typo'd `disable` can leave rules firing that the author thought were suppressed. `LX002` makes every malformed pragma visible.

```xml
<!-- non-compliant: 'disableonce' must be two words -->
<!-- xaml-lint disableonce LX100 -->
<Button Grid.Row="1" />
```

```xml
<!-- compliant -->
<!-- xaml-lint disable once LX100 -->
<Button Grid.Row="1" />
```

## How to fix violations

Check the grammar (spec §3.4). Valid directives:

- `<!-- xaml-lint disable RULE [RULE...] -->`
- `<!-- xaml-lint disable once RULE [RULE...] -->`
- `<!-- xaml-lint restore RULE [RULE...] -->`
- `<!-- xaml-lint disable All -->`

Keywords are case-sensitive. Rule IDs match `[A-Z]+\d+`.

## How to suppress violations

You don't want to suppress `LX002` — silencing malformed-pragma reporting means typos survive code review. If you have a compelling reason, set in `xaml-lint.config.json`:

```json
{ "rules": { "LX002": "off" } }
```
