# LX101: Grid.Column without matching ColumnDefinition

<!-- Upstream: RXT102. -->

## Cause

A child element of a `<Grid>` sets `Grid.Column="N"` where `N` is not a valid column index
for the enclosing Grid — that is, `N` is greater than or equal to the number of columns
the Grid declares. A Grid with no `<Grid.ColumnDefinitions>` (or no
`ColumnDefinitions="..."` shorthand) has one implicit column, index 0.

## Rule description

`Grid.Column` is an attached integer property that pins a child to a specific column of
its enclosing Grid. Valid values are in the half-open range `[0, columnCount)`. At runtime,
values out of range silently clamp to the last column, producing layouts that rarely match
author intent.

```xaml
<Grid xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation">
    <Grid.ColumnDefinitions>
        <ColumnDefinition />
        <ColumnDefinition />
    </Grid.ColumnDefinitions>
    <!-- non-compliant: Grid.Column="2" but only columns 0 and 1 exist -->
    <Button Grid.Column="2" Content="OK" />
</Grid>
```

```xaml
<Grid xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation">
    <Grid.ColumnDefinitions>
        <ColumnDefinition />
        <ColumnDefinition />
        <ColumnDefinition />
    </Grid.ColumnDefinitions>
    <!-- compliant: 3 columns defined, Grid.Column="2" is valid -->
    <Button Grid.Column="2" Content="OK" />
</Grid>
```

The check covers both attribute syntax (`<Button Grid.Column="2" />`) and element syntax
(`<Button><Grid.Column>2</Grid.Column></Button>`). The `ColumnDefinitions="*,Auto"`
shorthand attribute is recognised when the target framework supports it —
WinUI/UWP/MAUI/Avalonia/Uno always; WPF starting from .NET 10. Configure your project's
framework via the [`frameworkVersion`](../config-reference.md#frameworkversion) field in
`xaml-lint.config.json`; if you opt into a legacy WPF target, see [LX104](LX104.md) which
flags the shorthand attribute itself. Non-integer values (e.g., markup extensions) and
signed values (`Grid.Column="-1"`) are skipped.

## How to fix violations

Either add the missing `<ColumnDefinition>` entries (or extend the
`ColumnDefinitions="..."` shorthand) until `columnCount` covers the value you use, or
change the `Grid.Column` value to a column that already exists. If the element should not
participate in Grid placement at all, remove the `Grid.Column` attribute entirely.

## How to suppress violations

For a single element (the next `XElement` after the comment):

```xaml
<!-- xaml-lint disable once LX101 -->
<Button Grid.Column="5" />
```

For a block of markup:

```xaml
<!-- xaml-lint disable LX101 -->
<Button Grid.Column="5" />
<Button Grid.Column="6" />
<!-- xaml-lint restore LX101 -->
```

For a whole file or project, edit `xaml-lint.config.json`:

```json
{ "rules": { "LX101": "off" } }
```
