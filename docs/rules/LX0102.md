# LX0102: Grid.RowSpan exceeds available rows

<!-- Upstream: RXT103. -->

**Applies to:** `Avalonia`  `MAUI`  `Uno`  `UWP`  `WinUI 3`  `WPF`

## Cause

A child element of a `<Grid>` sets `Grid.RowSpan="N"` where `N` is greater than the total
number of rows the Grid declares. A Grid with no `<Grid.RowDefinitions>` (or no
`RowDefinitions="..."` shorthand) has one implicit row, so `Grid.RowSpan="2"` exceeds it.

## Rule description

`Grid.RowSpan` is an attached integer property that specifies how many rows a child
element occupies vertically. A span equal to the total row count is legal — it is the
common "header across all rows" idiom. A span greater than the total row count has no
runtime meaning beyond "span everything," and almost always indicates the author either
intended to add more rows or miscounted.

```xaml
<Grid>
    <Grid.RowDefinitions>
        <RowDefinition />
        <RowDefinition />
    </Grid.RowDefinitions>
    <!-- non-compliant: Grid.RowSpan="3" but only 2 rows exist -->
    <Rectangle Grid.RowSpan="3" Fill="LightGray" />
</Grid>
```

```xaml
<Grid>
    <Grid.RowDefinitions>
        <RowDefinition />
        <RowDefinition />
    </Grid.RowDefinitions>
    <!-- compliant: Grid.RowSpan="2" matches the row count; spans the whole Grid -->
    <Rectangle Grid.RowSpan="2" Fill="LightGray" />
</Grid>
```

The check considers the span value in isolation (`span > rowCount`) — it does not combine
`Grid.Row` with `Grid.RowSpan` to detect overflow from a non-zero starting row. The
combined overflow case (a non-zero `Grid.Row` that shifts the span past the end) is a
planned future check; today, [LX0100](LX0100.md) separately flags a starting row that is
itself out of range, independent of any span. Both attribute and element syntaxes are supported, and the
`RowDefinitions="Auto,*"` shorthand is honored when the target framework supports it
(WinUI/UWP/MAUI/Avalonia/Uno always; WPF on .NET 10+; configurable via
[`frameworkVersion`](../config-reference.md#frameworkversion); see also
[LX0104](LX0104.md)).

## How to fix violations

Either add more `<RowDefinition>` entries until the span fits, or reduce the `Grid.RowSpan`
value. If the element was meant to span everything, a value equal to the row count is the
idiomatic choice.

## How to suppress violations

For a single element (the next `XElement` after the comment):

```xaml
<!-- xaml-lint disable once LX0102 -->
<Rectangle Grid.RowSpan="99" />
```

For a block of markup:

```xaml
<!-- xaml-lint disable LX0102 -->
<Rectangle Grid.RowSpan="99" />
<Border Grid.RowSpan="99" />
<!-- xaml-lint restore LX0102 -->
```

For a whole file or project, edit `xaml-lint.config.json`:

```json
{ "rules": { "LX0102": "off" } }
```
