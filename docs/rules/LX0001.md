# LX0001: Malformed XAML

<!-- Upstream: RXT999. Emitted by the XML parser, not a rule class. -->

**Applies to:** `Avalonia`  `MAUI`  `Uno`  `UWP`  `WinUI 3`  `WPF`

## Cause

The XAML file could not be parsed as well-formed XML: unclosed tags, mismatched end tags, invalid entity references, or an encoding the parser could not decode.

## Rule description

`xaml-lint` parses every input with `System.Xml.Linq`. If parsing throws, no rules can run against that file and `LX0001` is emitted at the parser's reported line/column. The file is skipped for rule evaluation but counted toward the tool's exit code.

```xml
<!-- non-compliant: <Button> is opened but never closed before </Grid> -->
<Grid>
    <Button>
</Grid>
```

```xml
<!-- compliant -->
<Grid>
    <Button />
</Grid>
```

## How to fix violations

Open the file in an editor that surfaces XML errors (VS, VS Code with the XAML extension, Rider), locate the reported line, and close or balance the offending tag. If the file is intentionally a partial fragment, do not pass it to `xaml-lint` — pass only complete XAML documents.

## How to suppress violations

`LX0001` cannot be suppressed in-source (a malformed file can't host a `<!-- xaml-lint -->` pragma reliably). To silence it across a project, edit `xaml-lint.config.json`:

```json
{ "rules": { "LX0001": "off" } }
```

This is rarely the right call — parse errors almost always indicate real bugs.
