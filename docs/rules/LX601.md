# LX601: Line.Fill has no effect

<!-- Upstream: RXT320. -->

**Applies to:** `Avalonia`  `MAUI`  `Uno`  `UWP`  `WinUI 3`  `WPF`

## Cause

A `<Line>` element has a `Fill` attribute set.

## Rule description

A line is a one-dimensional shape; it has no interior surface to fill.
Setting `Fill` on a `Line` is inherited from the shared `Shape` base class
but does nothing at render time — true across every XAML dialect (WPF,
WinUI 3, UWP, .NET MAUI, Avalonia, Uno). The attribute is harmless but
misleading: future readers will expect a colored line and see no change,
leading to wasted debugging.

The rule fires on any `Fill` value (literal or bound); presence is the
signal.

```xaml
<!-- non-compliant: Fill does nothing on a Line -->
<Line X1="0" Y1="0" X2="100" Y2="0" Fill="Red" />
```

```xaml
<!-- compliant: Stroke is the right property for lines -->
<Line X1="0" Y1="0" X2="100" Y2="0" Stroke="Red" />
```

## How to fix violations

Delete the `Fill` attribute. If you wanted the line to be colored, set
`Stroke` (and, optionally, `StrokeThickness`) instead.

## How to suppress violations

For a single element:

```xaml
<!-- xaml-lint disable once LX601 -->
<Line Fill="Red" />
```

For a whole file or project:

```json
{ "rules": { "LX601": "off" } }
```
