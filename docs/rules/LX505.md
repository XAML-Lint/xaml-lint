# LX505: Pin lacks Label

<!-- Upstream: RXT325. -->

## Cause

A MAUI `<Pin>` element is declared without a `Label` attribute.

## Rule description

`Pin` is the marker type for `Microsoft.Maui.Controls.Maps.Map`. `Label`
is a required property — the control throws `ArgumentException` when a pin
without a label is added to the map. Static analysis cannot always see the
runtime attach path, so it's better to fail at author time.

Any `Label` value — literal or bound — suppresses the rule.

```xaml
<ContentPage xmlns="http://schemas.microsoft.com/dotnet/2021/maui">
    <!-- non-compliant: throws at runtime -->
    <Pin />
</ContentPage>
```

```xaml
<ContentPage xmlns="http://schemas.microsoft.com/dotnet/2021/maui">
    <!-- compliant -->
    <Pin Label="Home" />
</ContentPage>
```

## How to fix violations

Add a `Label` attribute. The value is used by the map control for the pin's
accessible name, info-window title, and hit-test string — make it a
user-meaningful description, not an internal identifier.

## How to suppress violations

For a single element:

```xaml
<!-- xaml-lint disable once LX505 -->
<Pin />
```

For a whole file or project:

```json
{ "rules": { "LX505": "off" } }
```
