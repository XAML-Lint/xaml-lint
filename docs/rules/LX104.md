# LX104: Grid definition shorthand not supported by target framework

<!-- xaml-lint-original; no upstream RXT equivalent. -->

## Cause

A `<Grid>` element uses the `RowDefinitions="Auto,*,..."` or `ColumnDefinitions="*,Auto"`
shorthand attribute, but the configured `dialect` + `frameworkVersion` combination does
not support that shorthand. Today this fires for **WPF on .NET 9 or earlier** — the
shorthand was added to WPF in .NET 10 ([dotnet/wpf#10866](https://github.com/dotnet/wpf/pull/10866));
WinUI/UWP/MAUI/Avalonia/Uno have always supported it.

## Rule description

The shorthand attribute is a quality-of-life feature: a `RowDefinitions="Auto,*"` reads
much more naturally than four lines of `<Grid.RowDefinitions><RowDefinition Height="Auto"/>...</Grid.RowDefinitions>`.
On runtimes that don't support it, however, the attribute is silently ignored — the Grid
behaves as if no row definitions were declared (one implicit row), and any `Grid.Row`
references past index 0 will silently clamp at runtime. The rule fires proactively so
the issue surfaces at lint time, not later.

```xaml
<!-- non-compliant: WPF .NET 9 ignores this attribute, leaving the Grid with 1 implicit row -->
<Grid xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
      RowDefinitions="Auto,*">
    <Button Grid.Row="1" />
</Grid>
```

```xaml
<!-- compliant: explicit element-syntax declarations work on every WPF version -->
<Grid xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation">
    <Grid.RowDefinitions>
        <RowDefinition Height="Auto" />
        <RowDefinition Height="*" />
    </Grid.RowDefinitions>
    <Button Grid.Row="1" />
</Grid>
```

The rule reads the `frameworkVersion` field from `xaml-lint.config.json` (root or
per-override). If `frameworkVersion` is omitted, the rule assumes the newest framework
and never fires — so the default is "trust the user's modern toolchain." Set
`"frameworkVersion": "9"` on a legacy-WPF project to opt into the proactive check; bump
to `"10"` (or remove the field) when you migrate.

## How to fix violations

Either:

- Convert the shorthand to the equivalent element syntax (the example above), or
- Bump the project to .NET 10 (where WPF supports the shorthand natively) and update
  `frameworkVersion` in your `xaml-lint.config.json`.

## How to suppress violations

For a single Grid (the next `XElement` after the comment):

```xaml
<!-- xaml-lint disable once LX104 -->
<Grid RowDefinitions="Auto,*" />
```

For a block of markup:

```xaml
<!-- xaml-lint disable LX104 -->
<Grid RowDefinitions="Auto,*" />
<Grid ColumnDefinitions="*,Auto" />
<!-- xaml-lint restore LX104 -->
```

For a whole file or project, edit `xaml-lint.config.json`:

```json
{ "rules": { "LX104": "off" } }
```
