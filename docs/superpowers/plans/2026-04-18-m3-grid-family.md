**Goal:** Ship the four Grid-family lint rules on top of M2's plumbing: `LX100` (Grid.Row without matching RowDefinition), `LX101` (Grid.Column without matching ColumnDefinition), `LX102` (Grid.RowSpan exceeds available rows), `LX103` (Grid.ColumnSpan exceeds available columns). Tag **v0.3.0** with docs, presets, schema, AnalyzerReleases, and the RXT comparison all regenerated and in sync.

**Architecture:** All four rules share one ancestry-walking helper and one attached-property reader. Each rule's `Analyze` is a short descendant loop that delegates to the helpers and produces a diagnostic on the offending `Grid.Row` / `Grid.Column` / `Grid.RowSpan` / `Grid.ColumnSpan` source (attribute or element). The four rules are independent: each lands as its own commit with tests and an AnalyzerReleases.Unshipped row, so a reviewer can bisect if the helper misbehaves.

**Tech Stack:** .NET 10, `System.Xml.Linq` (`XDocument` with `LoadOptions.SetLineInfo | LoadOptions.PreserveWhitespace` from M1), the M1 source generator (`XamlLint.Core.SourceGen`), the M1 `XamlLint.DocTool` (stubs docs, writes presets, writes schema; `--check` in CI), xUnit v3 + Microsoft Testing Platform + AwesomeAssertions for tests using the M1 `XamlDiagnosticVerifier<TRule>` marker harness, `Nerdbank.GitVersioning` for the `0.2-alpha` → `v0.3.0` bump.

## Notes before starting

**Test harness reminders (from M2).**

1. `XamlDiagnosticVerifier<TRule>.Analyze(markedSource)` loads the marked string, strips `[|…|]` / `{|…|}` markers, and asserts the rule's diagnostic spans match exactly what's marked. `[|…|]` is shorthand for "expect this rule's diagnostic here"; `{|LXNNN:…|}` spells out the rule ID (useful for multi-rule assertions — not needed in M3 since each test file pins one rule). `{|LXNNN|}` (span-less) means "expect any diagnostic for this rule somewhere in the source" — handy when the diagnostic span isn't worth asserting precisely (used once in M3 for element-syntax attached properties).

2. The verifier loads with `Dialect.Wpf` by default. An overload exists: `Analyze(markedSource, Dialect.Maui)` for dialect-gated tests. All four M3 rules apply to `Dialect.All`, so the default is fine for every M3 test.

3. Raw triple-quoted strings (`""" ... """`) keep the markers aligned to the source columns. Do not replace with regular `"..."` strings: the column-math in the harness relies on the raw layout.

**Source-generator contract.** Declaring a `public sealed partial class LXnnn_Foo : IXamlRule` with a `[XamlRule(...)]` attribute is sufficient to have it picked up by `XamlLint.Core.SourceGen` at build time. The generator emits a `Metadata` property and adds the class to `GeneratedRuleCatalog.Rules`. No manual registration.

**Tool rules vs lint rules.** Tool rules (LX001–LX006) implement `IToolRule` (a marker extension of `IXamlRule`) and return `Enumerable.Empty<Diagnostic>()` from `Analyze`. The dispatcher skips them in its constructor (`_rules = rules.Where(r => r is not IToolRule).ToArray()`). M3 rules must **not** implement `IToolRule`.

**Package versions.** M3 does not add any NuGet packages. Central package management in `Directory.Packages.props` already pins every dependency needed.

**Test runner invocation.** `dotnet test --solution D:/GitHub/jizc/xaml-lint/xaml-lint.slnx` (MTP ignores `dotnet test` positional paths; `--solution` is mandatory on .NET 10). For a filtered run: `dotnet test --solution … --filter-method "*TestName*"`.

**Existing files that M3 must not destroy.** Every rule file under `src/XamlLint.Core/Rules/Tool|Bindings|Naming|Resources/`, every doc file under `docs/rules/` (LX001.md–LX006.md, LX200.md, LX300.md, LX400.md, tool.md, bindings.md, naming.md, resources.md, comparison-with-rapid-xaml-toolkit.md), `AnalyzerReleases.Shipped.md`'s `## Release 0.1.0` and `## Release 0.2.0` sections, `CHANGELOG.md`, the `schema/v1/presets/*.json` (these get regenerated by DocTool — do not hand-edit), the M2 helpers (`XamlNamespaces.cs`, `LocationHelpers.cs`, `MarkupExtensionHelpers.cs`), the M1 test harness under `tests/XamlLint.Core.Tests/TestInfrastructure/`.

**Rule responsibilities (one-liners):**

- `LX100_GridRowWithoutDefinition` — flags `Grid.Row="N"` on a Grid child when `N >= rowCount` (no matching RowDefinition).
- `LX101_GridColumnWithoutDefinition` — flags `Grid.Column="N"` on a Grid child when `N >= columnCount`.
- `LX102_GridRowSpanExceedsRows` — flags `Grid.RowSpan="N"` on a Grid child when `N > rowCount` (RXT semantics — span alone exceeds total rows, independent of the child's starting row).
- `LX103_GridColumnSpanExceedsColumns` — flags `Grid.ColumnSpan="N"` on a Grid child when `N > columnCount`.

**Helper responsibilities (one-liners):**

- `GridAncestryHelpers.FindNearestGridAncestor(XElement)` — walks `.Parent` chain, returns the first element whose `Name.LocalName == "Grid"` (dialect-agnostic — every target dialect names the type `Grid`), or `null` if none.
- `GridAncestryHelpers.CountRowDefinitions(XElement grid)` / `CountColumnDefinitions(XElement grid)` — returns the number of rows/columns the Grid declares. Checks the WinUI/UWP shorthand `RowDefinitions="Auto,*,..."` / `ColumnDefinitions="..."` attribute first; falls back to counting `<RowDefinition>` / `<ColumnDefinition>` children inside the `<Grid.RowDefinitions>` / `<Grid.ColumnDefinitions>` property element. Returns `1` (the implicit single row/column) when neither form is present.
- `GridAncestryHelpers.TryReadIntegerAttachedProperty(XElement, string propertyName)` — returns the integer value of an attached property in either attribute syntax (`<Button Grid.Row="1" />`) or element syntax (`<Button><Grid.Row>1</Grid.Row></Button>`), plus the `XObject` source for diagnostic spans. Returns `null` when the property is absent or when the value does not parse as an integer (e.g., `Grid.Row="{Binding Idx}"`).
- `LocationHelpers.GetElementNameSpan(XElement)` — returns the 1-based `(StartLine, StartCol, EndLine, EndCol)` span covering just the element's opening-tag name (e.g., `Grid.Row` inside `<Grid.Row>`). Required because element-syntax attached properties use an `XElement` source, not an `XAttribute`.

```csharp
using System.Globalization;
using System.Xml.Linq;

namespace XamlLint.Core.Helpers;

/// <summary>
/// Grid-layout introspection for LX100–LX103. Walks the XML tree to find the nearest
/// <c>&lt;Grid&gt;</c> ancestor, counts its declared rows/columns (both element-syntax and
/// the WinUI/UWP <c>RowDefinitions="Auto,*,..."</c> shorthand), and reads integer attached
/// properties like <c>Grid.Row</c> in either attribute or element syntax.
/// </summary>
public static class GridAncestryHelpers
{
    public const string GridElementName = "Grid";
    public const string RowDefinitionsPropertyElement = "Grid.RowDefinitions";
    public const string ColumnDefinitionsPropertyElement = "Grid.ColumnDefinitions";
    public const string RowDefinitionElement = "RowDefinition";
    public const string ColumnDefinitionElement = "ColumnDefinition";
    public const string RowDefinitionsShorthandAttribute = "RowDefinitions";
    public const string ColumnDefinitionsShorthandAttribute = "ColumnDefinitions";

    /// <summary>
    /// Value carrier for <see cref="TryReadIntegerAttachedProperty"/>. <paramref name="Source"/>
    /// is either an <see cref="XAttribute"/> (attribute syntax) or an <see cref="XElement"/>
    /// (element syntax) that the caller can hand to <c>LocationHelpers</c> to compute a span.
    /// </summary>
    public readonly record struct AttachedPropertyValue(int Value, XObject Source);

    /// <summary>
    /// Returns the first ancestor whose <c>Name.LocalName == "Grid"</c>, or <c>null</c>. The
    /// check is by local name only — every target dialect names the type <c>Grid</c> and none
    /// carries a dialect-specific prefix.
    /// </summary>
    public static XElement? FindNearestGridAncestor(XElement element)
    {
        for (var ancestor = element.Parent; ancestor is not null; ancestor = ancestor.Parent)
        {
            if (ancestor.Name.LocalName == GridElementName)
                return ancestor;
        }
        return null;
    }

    /// <summary>
    /// Returns the number of rows the Grid declares. The WinUI/UWP shorthand attribute
    /// <c>RowDefinitions="Auto,*,..."</c> takes precedence; falls back to counting
    /// <c>&lt;RowDefinition&gt;</c> children inside <c>&lt;Grid.RowDefinitions&gt;</c>. A Grid
    /// that declares neither has an implicit single row — returns 1.
    /// </summary>
    public static int CountRowDefinitions(XElement grid) =>
        CountDefinitions(grid, RowDefinitionsShorthandAttribute, RowDefinitionsPropertyElement, RowDefinitionElement);

    /// <summary>
    /// Mirror of <see cref="CountRowDefinitions"/> for columns.
    /// </summary>
    public static int CountColumnDefinitions(XElement grid) =>
        CountDefinitions(grid, ColumnDefinitionsShorthandAttribute, ColumnDefinitionsPropertyElement, ColumnDefinitionElement);

    private static int CountDefinitions(
        XElement grid,
        string shorthandAttributeName,
        string propertyElementName,
        string definitionElementName)
    {
        var shorthand = grid.Attribute(shorthandAttributeName);
        if (shorthand is not null)
            return CountCommaSeparated(shorthand.Value);

        var propertyElement = grid.Elements().FirstOrDefault(
            e => e.Name.LocalName == propertyElementName);
        if (propertyElement is null)
            return 1;  // implicit single row/column

        return propertyElement.Elements().Count(
            e => e.Name.LocalName == definitionElementName);
    }

    /// <summary>
    /// Reads an integer attached property (e.g. <c>Grid.Row</c>) in either attribute syntax
    /// (<c>&lt;Button Grid.Row="1" /&gt;</c>) or element syntax
    /// (<c>&lt;Button&gt;&lt;Grid.Row&gt;1&lt;/Grid.Row&gt;&lt;/Button&gt;</c>). Returns the
    /// integer value plus the source <see cref="XObject"/> for span computation, or
    /// <c>null</c> when the property is absent or its value does not parse as an integer
    /// (e.g. a markup-extension value like <c>{Binding Idx}</c>).
    /// </summary>
    public static AttachedPropertyValue? TryReadIntegerAttachedProperty(
        XElement element,
        string propertyName)
    {
        var attr = element.Attribute(propertyName);
        if (attr is not null)
        {
            return int.TryParse(attr.Value, NumberStyles.Integer, CultureInfo.InvariantCulture, out var attrValue)
                ? new AttachedPropertyValue(attrValue, attr)
                : null;
        }

        var elementChild = element.Elements().FirstOrDefault(e => e.Name.LocalName == propertyName);
        if (elementChild is not null)
        {
            return int.TryParse(elementChild.Value.Trim(), NumberStyles.Integer, CultureInfo.InvariantCulture, out var elemValue)
                ? new AttachedPropertyValue(elemValue, elementChild)
                : null;
        }

        return null;
    }

    private static int CountCommaSeparated(string value)
    {
        if (string.IsNullOrWhiteSpace(value)) return 0;
        var count = 0;
        foreach (var part in value.Split(','))
            if (!string.IsNullOrWhiteSpace(part)) count++;
        return count;
    }
}
```

```csharp
/// <summary>
/// Returns the 1-based span that covers an element's opening-tag name only (for example,
/// <c>Grid.Row</c> inside <c>&lt;Grid.Row&gt;</c>). Used by rules whose diagnostic source is
/// an <see cref="XElement"/> — element-syntax attached properties. Line info on
/// <see cref="XElement"/> points at the first character of the name, immediately after the
/// opening <c>&lt;</c>. The returned <c>EndCol</c> is one past the last character of the
/// local name (exclusive), consistent with <see cref="GetAttributeSpan"/>.
/// </summary>
public static (int StartLine, int StartCol, int EndLine, int EndCol) GetElementNameSpan(
    XElement element)
{
    var lineInfo = (IXmlLineInfo)element;
    if (!lineInfo.HasLineInfo())
        throw new InvalidOperationException(
            "Element has no line info; XamlDocument must load with LoadOptions.SetLineInfo.");

    var line = lineInfo.LineNumber;
    var startCol = lineInfo.LinePosition;
    var nameLen = element.Name.LocalName.Length;
    return (line, startCol, line, startCol + nameLen);
}
```

```csharp
using System.Xml.Linq;
using XamlLint.Core.Helpers;

namespace XamlLint.Core.Rules.Layout;

[XamlRule(
    Id = "LX100",
    UpstreamId = "RXT101",
    Title = "Grid.Row without matching RowDefinition",
    DefaultSeverity = Severity.Warning,
    Dialects = Dialect.All,
    HelpUri = "https://github.com/jizc/xaml-lint/blob/main/docs/rules/LX100.md")]
public sealed partial class LX100_GridRowWithoutDefinition : IXamlRule
{
    public IEnumerable<Diagnostic> Analyze(XamlDocument document, RuleContext context)
    {
        if (document.Root is null) yield break;

        foreach (var element in document.Root.DescendantsAndSelf())
        {
            // Skip property-element pseudo-nodes like <Grid.RowDefinitions>; they are not
            // layout children and never carry attached properties.
            if (element.Name.LocalName.Contains('.')) continue;

            var read = GridAncestryHelpers.TryReadIntegerAttachedProperty(element, "Grid.Row");
            if (read is null) continue;

            var grid = GridAncestryHelpers.FindNearestGridAncestor(element);
            if (grid is null) continue;

            var rowCount = GridAncestryHelpers.CountRowDefinitions(grid);
            var rowValue = read.Value.Value;
            if (rowValue < rowCount) continue;

            var span = read.Value.Source switch
            {
                XAttribute attr => LocationHelpers.GetAttributeSpan(attr, context.Source),
                XElement el => LocationHelpers.GetElementNameSpan(el),
                _ => throw new InvalidOperationException(
                    $"Unexpected attached-property source type {read.Value.Source.GetType().Name}.")
            };

            yield return new Diagnostic(
                RuleId: Metadata.Id,
                Severity: Metadata.DefaultSeverity,
                Message: $"Grid.Row=\"{rowValue}\" but the enclosing Grid declares only {rowCount} row{(rowCount == 1 ? "" : "s")}.",
                File: document.FilePath,
                StartLine: span.StartLine,
                StartCol: span.StartCol,
                EndLine: span.EndLine,
                EndCol: span.EndCol,
                HelpUri: Metadata.HelpUri);
        }
    }
}
```

```csharp
using XamlLint.Core.Helpers;

namespace XamlLint.Core.Rules.Layout;

[XamlRule(
    Id = "LX103",
    UpstreamId = "RXT104",
    Title = "Grid.ColumnSpan exceeds available columns",
    DefaultSeverity = Severity.Warning,
    Dialects = Dialect.All,
    HelpUri = "https://github.com/jizc/xaml-lint/blob/main/docs/rules/LX103.md")]
public sealed partial class LX103_GridColumnSpanExceedsColumns : IXamlRule
{
    public IEnumerable<Diagnostic> Analyze(XamlDocument document, RuleContext context)
    {
        if (document.Root is null) yield break;

        foreach (var element in document.Root.DescendantsAndSelf())
        {
            if (element.Name.LocalName.Contains('.')) continue;

            var read = GridAncestryHelpers.TryReadIntegerAttachedProperty(element, "Grid.ColumnSpan");
            if (read is null) continue;

            var grid = GridAncestryHelpers.FindNearestGridAncestor(element);
            if (grid is null) continue;

            var columnCount = GridAncestryHelpers.CountColumnDefinitions(grid);
            var spanValue = read.Value.Value;
            if (spanValue <= columnCount) continue;

            var span = read.Value.Source switch
            {
                XAttribute attr => LocationHelpers.GetAttributeSpan(attr, context.Source),
                XElement el => LocationHelpers.GetElementNameSpan(el),
                _ => throw new InvalidOperationException(
                    $"Unexpected attached-property source type {read.Value.Source.GetType().Name}.")
            };

            yield return new Diagnostic(
                RuleId: Metadata.Id,
                Severity: Metadata.DefaultSeverity,
                Message: $"Grid.ColumnSpan=\"{spanValue}\" exceeds the enclosing Grid's {columnCount} column{(columnCount == 1 ? "" : "s")}.",
                File: document.FilePath,
                StartLine: span.StartLine,
                StartCol: span.StartCol,
                EndLine: span.EndLine,
                EndCol: span.EndCol,
                HelpUri: Metadata.HelpUri);
        }
    }
}
```

````markdown
# LX100: Grid.Row without matching RowDefinition

<!-- Upstream: RXT101. -->

## Cause

A child element of a `<Grid>` sets `Grid.Row="N"` where `N` is not a valid row index for
the enclosing Grid — that is, `N` is greater than or equal to the number of rows the Grid
declares. A Grid with no `<Grid.RowDefinitions>` (or no `RowDefinitions="..."` shorthand)
has one implicit row, index 0.

## Rule description

`Grid.Row` is an attached integer property that pins a child to a specific row of its
enclosing Grid. Valid values are in the half-open range `[0, rowCount)`. At runtime, values
out of range silently clamp to the last row, which almost always produces a layout that
differs from what the author intended — two controls stacking on the final row instead of
one per row, for example.

```xaml
<Grid xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation">
    <Grid.RowDefinitions>
        <RowDefinition />
        <RowDefinition />
    </Grid.RowDefinitions>
    <!-- non-compliant: Grid.Row="2" but only rows 0 and 1 exist -->
    <Button Grid.Row="2" Content="OK" />
</Grid>
```

```xaml
<Grid xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation">
    <Grid.RowDefinitions>
        <RowDefinition />
        <RowDefinition />
        <RowDefinition />
    </Grid.RowDefinitions>
    <!-- compliant: 3 rows defined, Grid.Row="2" is valid -->
    <Button Grid.Row="2" Content="OK" />
</Grid>
```

The check covers both attribute syntax (`<Button Grid.Row="2" />`) and element syntax
(`<Button><Grid.Row>2</Grid.Row></Button>`). The WinUI/UWP shorthand
`RowDefinitions="Auto,*,Auto"` is also recognised. Values that do not parse as integers
(for example `Grid.Row="{Binding Idx}"`) are skipped — the rule only fires on static
values it can reason about.

## How to fix violations

Either add the missing `<RowDefinition>` entries (or extend the `RowDefinitions="..."`
shorthand) until `rowCount` covers the value you use, or change the `Grid.Row` value to a
row that already exists. If the element should not participate in Grid placement at all,
remove the `Grid.Row` attribute entirely.

## How to suppress violations

For a single element (the next `XElement` after the comment):

```xaml
<!-- xaml-lint disable once LX100 -->
<Button Grid.Row="5" />
```

For a block of markup:

```xaml
<!-- xaml-lint disable LX100 -->
<Button Grid.Row="5" />
<Button Grid.Row="6" />
<!-- xaml-lint restore LX100 -->
```

For a whole file or project, edit `xaml-lint.config.json`:

```json
{ "rules": { "LX100": "off" } }
```
````

````markdown
# LX102: Grid.RowSpan exceeds available rows

<!-- Upstream: RXT103. -->

## Cause

A child element of a `<Grid>` sets `Grid.RowSpan="N"` where `N` is greater than the total
number of rows the Grid declares. A Grid with no `<Grid.RowDefinitions>` (or no
`RowDefinitions="..."` shorthand) has one implicit row, so `Grid.RowSpan="2"` exceeds it.

## Rule description

`Grid.RowSpan` is an attached integer property that specifies how many rows a child
element occupies vertically. A span equal to the total row count is legal — it is the
common "header across all rows" idiom. A span greater than the total row count has no
runtime meaning beyond "span everything," and almost always indicates the author either
intended to add more rows or miscounted.

```xaml
<Grid xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation">
    <Grid.RowDefinitions>
        <RowDefinition />
        <RowDefinition />
    </Grid.RowDefinitions>
    <!-- non-compliant: Grid.RowSpan="3" but only 2 rows exist -->
    <Rectangle Grid.RowSpan="3" Fill="LightGray" />
</Grid>
```

```xaml
<Grid xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation">
    <Grid.RowDefinitions>
        <RowDefinition />
        <RowDefinition />
    </Grid.RowDefinitions>
    <!-- compliant: Grid.RowSpan="2" matches the row count; spans the whole Grid -->
    <Rectangle Grid.RowSpan="2" Fill="LightGray" />
</Grid>
```

The check considers the span value in isolation (`span > rowCount`) — it does not combine
`Grid.Row` with `Grid.RowSpan` to detect overflow from a non-zero starting row. That
stricter overflow check is out of scope for this rule; an element starting past the last
row is flagged separately by [LX100](LX100.md). Both attribute and element syntaxes are
supported, and the WinUI/UWP `RowDefinitions="Auto,*"` shorthand is honored.

## How to fix violations

Either add more `<RowDefinition>` entries until the span fits, or reduce the `Grid.RowSpan`
value. If the element was meant to span everything, a value equal to the row count is the
idiomatic choice.

## How to suppress violations

For a single element (the next `XElement` after the comment):

```xaml
<!-- xaml-lint disable once LX102 -->
<Rectangle Grid.RowSpan="99" />
```

For a block of markup:

```xaml
<!-- xaml-lint disable LX102 -->
<Rectangle Grid.RowSpan="99" />
<Border Grid.RowSpan="99" />
<!-- xaml-lint restore LX102 -->
```

For a whole file or project, edit `xaml-lint.config.json`:

```json
{ "rules": { "LX102": "off" } }
```
````
