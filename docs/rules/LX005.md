# LX005: Skipping non-XAML file

<!-- Emitted by the file enumerator when --force is not set. -->

## Cause

A path was passed to `xaml-lint` whose extension isn't `.xaml`.

## Rule description

`xaml-lint` won't attempt to parse arbitrary text as XAML by default — that would produce confusing `LX001` diagnostics on unrelated files. Instead it skips non-`.xaml` files and emits `LX005` at `info`, which does not affect the exit code.

Pass `--force` to lint files regardless of extension (useful for `.paml`, `.axaml`, or custom extensions).

## How to fix violations

- Rename the file to `.xaml` if it really is XAML.
- Or pass `--force` to include it.
- Or exclude the file with `--exclude` to stop being notified.

## How to suppress violations

```json
{ "rules": { "LX005": "off" } }
```

If you never want to see this notice, set it to `off` in your project config.
