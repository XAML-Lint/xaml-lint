# LX0402: Image Source filename invalid on Android

<!-- Upstream: RXT310. -->

**Applies to:** `MAUI`

## Cause

A MAUI `<Image>` has its `Source` set to a literal local filename whose
leaf segment violates Android drawable naming rules.

## Rule description

Android's build pipeline copies image assets into `res/drawable/`, which
enforces strict naming: **lowercase letters, digits, underscores, and
periods only**, and **the name cannot start with a digit**. A filename
that breaks those rules will fail the Android build of a cross-platform
MAUI project, often with an error that points at the resource stage
rather than at the offending XAML — hard to diagnose.

The rule validates only the leaf segment of the path (the characters after
the last `/` or `\`). Subfolder names and drive letters are not checked
because they are part of the MAUI project layout, not the Android drawable
name.

**Values that suppress the rule (xaml-lint-specific — see
`comparison-with-rapid-xaml-toolkit.md`):**

- A markup extension (`{Binding …}`, `{StaticResource …}`, …).
- A URI starting with `http://`, `https://`, `ms-appx:`, `ms-appdata:`,
  or `file://`. These are not local files and don't flow through the
  Android drawable pipeline.

```xaml
<!-- non-compliant: uppercase M; hyphen -->
<Image Source="MyIcon.png" />
<Image Source="my-icon.png" />

<!-- non-compliant: starts with a digit -->
<Image Source="2x_icon.png" />
```

```xaml
<!-- compliant -->
<Image Source="my_icon.png" />
<Image Source="icon_2x.png" />
<Image Source="Resources/my_icon.png" />  <!-- folder path unchecked; leaf OK -->
```

## How to fix violations

Rename the file to use lowercase letters, digits, underscores, and periods
only, and make sure it does not start with a digit. Update both the
project's asset tree and the `Source` reference. If the image is
platform-specific (e.g., an iOS-only PDF), move it out of the shared
`Resources/Images` path.

## How to suppress violations

For a single element:

```xaml
<!-- xaml-lint disable once LX0402 -->
<Image Source="MyIcon.png" />
```

For a whole file or project:

```json
{ "rules": { "LX0402": "off" } }
```
