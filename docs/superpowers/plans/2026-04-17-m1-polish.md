## Task 1: PragmaParser — require token boundary after `xaml-lint`

**Bug:** `StartsWith("xaml-lint", Ordinal)` classifies `<!-- xaml-linting foo -->` or `<!-- xaml-lint-bla -->` as a malformed pragma and emits `LX002`. Should ignore non-pragma comments entirely.

**File:** `src/XamlLint.Core/Suppressions/PragmaParser.cs`

Replace the `body.StartsWith("xaml-lint", …)` check at the top of the `foreach (var comment in comments)` loop with a token-boundary guard:

```csharp
// Accept "xaml-lint" followed by whitespace or end-of-string; reject "xaml-linting", "xaml-lint-foo", etc.
if (!(body == "xaml-lint" || (body.StartsWith("xaml-lint", StringComparison.Ordinal) && body.Length > 9 && char.IsWhiteSpace(body[9]))))
    continue;
```

**Test:** add to `PragmaParserTest.cs`:

```csharp
[Fact]
public void Comment_with_xaml_lint_prefix_but_no_boundary_is_not_a_pragma()
{
    const string src = """
        <Root xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation">
            <!-- xaml-linting is a word, not a directive -->
            <!-- xaml-lint-something -->
            <A />
        </Root>
        """;
    var doc = Parse(src);
    var result = PragmaParser.Parse(doc);

    result.Diagnostics.Should().BeEmpty();
    result.Map.IsEmpty.Should().BeTrue();
}
```

**Commit:** `Fix PragmaParser: only treat 'xaml-lint' as pragma when followed by whitespace`

## Task 2: PragmaParser — accept any whitespace as a separator

**Bug:** `body.Split(' ', StringSplitOptions.RemoveEmptyEntries)` only splits on space. Tabs in pragmas (e.g. `<!-- xaml-lint\tdisable\tLX100 -->`) fail to parse and emit LX002. Spec §3.4 says "whitespace is flexible."

**File:** `src/XamlLint.Core/Suppressions/PragmaParser.cs`

In `TryParseDirective`:

```csharp
// before:
var parts = body.Split(' ', StringSplitOptions.RemoveEmptyEntries);
// after:
var parts = body.Split((char[]?)null, StringSplitOptions.RemoveEmptyEntries);
```

(`Split(null, ...)` splits on all Unicode whitespace.)

**Test:** add to `PragmaParserTest.cs`:

```csharp
[Fact]
public void Tabs_between_tokens_are_accepted()
{
    const string src = "<Root xmlns=\"http://schemas.microsoft.com/winfx/2006/xaml/presentation\">\n" +
                       "\t<!--\txaml-lint\tdisable\tLX100\t-->\n" +
                       "\t<A />\n" +
                       "</Root>";
    var doc = Parse(src);
    var result = PragmaParser.Parse(doc);

    result.Diagnostics.Should().BeEmpty();
    result.Map.IsSuppressed("LX100", line: 3).Should().BeTrue();
}
```

**Commit:** `Fix PragmaParser: accept any whitespace between tokens`

## Task 3: PragmaParser — `disable once` covers whole multi-line element

**Bug:** `ComputeElementEndLine` walks descendants via `IXmlLineInfo.LineNumber`, which returns the **start-tag** line. Closing tags on their own line aren't visible, so for

```xml
<!-- xaml-lint disable once LX100 -->
<A>
    <B />
</A>
```

a diagnostic fired on the `</A>` line won't be suppressed.

**File:** `src/XamlLint.Core/Suppressions/PragmaParser.cs`

`ComputeElementEndLine` needs the *closing*-tag line. Two approaches:
- **(A) Use the next sibling's start line minus 1** — doesn't work for last children.
- **(B) Scan the raw source** — find `</LocalName>` starting from the element's start position.

Go with (B). Add a `source` parameter threaded through `Parse` and `ComputeElementEndLine`:

```csharp
// Parse signature stays the same; inside Parse, pass document.Source down.
// In the disable-once case:
var elementEnd = ComputeElementEndLine(nextElement, document.Source);

// New ComputeElementEndLine:
private static int ComputeElementEndLine(XElement el, string source)
{
    var info = (IXmlLineInfo)el;
    var startLine = info.LineNumber;
    var closingTag = $"</{el.Name.LocalName}>";

    // Walk the source from the element's starting line to find the closing tag.
    var idx = NthNewlineIndex(source, startLine - 1);      // start of el's start line
    var closeIdx = source.IndexOf(closingTag, idx, StringComparison.Ordinal);
    if (closeIdx < 0)
    {
        // Self-closing or couldn't locate; fall back to descendant max.
        var max = startLine;
        foreach (var n in el.DescendantNodesAndSelf())
            if (n is IXmlLineInfo li && li.HasLineInfo() && li.LineNumber > max) max = li.LineNumber;
        return max;
    }
    return LineAtOffset(source, closeIdx);
}

private static int NthNewlineIndex(string s, int n)
{
    if (n <= 0) return 0;
    var count = 0; var i = 0;
    while (i < s.Length && count < n)
    {
        if (s[i] == '\n') count++;
        i++;
    }
    return i;
}

private static int LineAtOffset(string s, int offset)
{
    var line = 1;
    for (var i = 0; i < offset && i < s.Length; i++)
        if (s[i] == '\n') line++;
    return line;
}
```

**Test:** add to `PragmaParserTest.cs`:

```csharp
[Fact]
public void Disable_once_covers_multi_line_element_including_closing_tag()
{
    const string src = """
        <Root xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation">
            <!-- xaml-lint disable once LX100 -->
            <A>
                <B />
            </A>
            <C />
        </Root>
        """;
    var doc = Parse(src);
    var result = PragmaParser.Parse(doc);

    result.Map.IsSuppressed("LX100", line: 3).Should().BeTrue();  // <A>
    result.Map.IsSuppressed("LX100", line: 4).Should().BeTrue();  // <B />
    result.Map.IsSuppressed("LX100", line: 5).Should().BeTrue();  // </A>
    result.Map.IsSuppressed("LX100", line: 6).Should().BeFalse(); // <C />
}
```

**Commit:** `Fix PragmaParser: disable once spans full multi-line element including closing tag`
