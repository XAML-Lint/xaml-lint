# LX0602: MAUI Shell nav-surface lacks Title and Icon

**Applies to:** `MAUI`

## Cause

A MAUI Shell navigation surface (`<Tab>`, `<ShellContent>`, `<FlyoutItem>`,
or `<MenuItem>`) carries neither a `Title` nor an `Icon` attribute. The
runtime renders the resulting tab strip, flyout entry, or menu row blank
â€” there is nothing for the user to read or recognise, so the destination
is unreachable through the navigation chrome.

## Rule description

Shell navigation chrome is identified visually by either a label
(`Title`) or a glyph (`Icon`). Either alone is enough â€” text-only nav
patterns (a flyout that lists "Home / Settings / About" with no icons)
are valid, and icon-only patterns (a compact bottom-tab bar with five
glyph-only tabs) are valid. What's not valid is omitting both, because
the runtime then has nothing to draw and the user has no way to spot or
target the entry.

The rule covers the four built-in Shell nav-surface element types:

| Element | Where it lives |
|---|---|
| `Tab` | Inside `<TabBar>` or as a direct `<Shell>` child |
| `ShellContent` | Inside `<Tab>` or `<TabBar>` |
| `FlyoutItem` | Direct `<Shell>` child rendered in the flyout |
| `MenuItem` | Direct `<Shell>` child rendered as a flyout footer item |

An attribute counts as "set" when its value is non-empty after trimming.
Bound values (`Title="{Binding HomeTitle}"`, `Icon="{StaticResource HomeGlyph}"`)
count as set â€” the literal attribute value is non-empty, so the rule
treats the author as having stated intent and does not fire. Empty
strings (`Title=""`) and whitespace-only values are treated as missing.

```xaml
<!-- non-compliant: nothing for the runtime to render -->
<Shell xmlns="http://schemas.microsoft.com/dotnet/2021/maui">
    <TabBar>
        <ShellContent ContentTemplate="{DataTemplate views:HomePage}" />
    </TabBar>
</Shell>
```

```xaml
<!-- compliant: text-only tab -->
<ShellContent Title="Home"
              ContentTemplate="{DataTemplate views:HomePage}" />

<!-- compliant: icon-only tab -->
<ShellContent Icon="home.png"
              ContentTemplate="{DataTemplate views:HomePage}" />

<!-- compliant: bound title -->
<ShellContent Title="{Binding HomeTitle}"
              ContentTemplate="{DataTemplate views:HomePage}" />
```

## What is out of scope

- **`BaseShellItem`-derived custom Shell items.** Recognition is keyed
  off the four built-in element local names (`Tab`, `ShellContent`,
  `FlyoutItem`, `MenuItem`); a custom `<local:HomeShellItem/>` does not
  fire the rule. Adding open-ended subclass detection would require a
  project-level type list and is deferred.
- **`Title`/`Icon` set via implicit Style.** The rule reads the
  attribute on the element itself and does not follow `Style` references
  (implicit or explicit) to inspect setters. A nav surface that inherits
  its label or glyph from a project-level style will still fire â€” pin to
  `:off` via pragma or config if your project relies on this idiom.
- **Element-form `<ShellContent.Title>` / `<ShellContent.Icon>`.** Only
  the attribute form is inspected. Property-element syntax for these
  values is uncommon for Shell nav and out of scope at v1.2.
- **`FlyoutDisplayOptions` / template-driven flyouts.** A `<FlyoutItem>`
  whose visible chrome is replaced by an `ItemTemplate` is still flagged
  if the item itself omits `Title` and `Icon` â€” the rule does not parse
  the template to discover whether it produces a label or glyph.

## How to fix violations

Set at least one of `Title` or `Icon`:

```xaml
<ShellContent Title="Home"
              Icon="home.png"
              ContentTemplate="{DataTemplate views:HomePage}" />
```

Pick the form that fits the navigation chrome you're rendering. A
bottom-tab bar typically uses both; a side flyout often uses `Title`
alone; a compact tab strip on a small-screen layout often uses `Icon`
alone.

## How to suppress violations

For a single element:

```xaml
<!-- xaml-lint disable once LX0602 -->
<ShellContent ContentTemplate="{DataTemplate views:HomePage}" />
```

For a block:

```xaml
<!-- xaml-lint disable LX0602 -->
<ShellContent ContentTemplate="{DataTemplate views:HomePage}" />
<ShellContent ContentTemplate="{DataTemplate views:SettingsPage}" />
<!-- xaml-lint restore LX0602 -->
```

For a whole file or project:

```json
{ "rules": { "LX0602": "off" } }
```
