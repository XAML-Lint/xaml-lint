# LX506: Slider sets both ThumbColor and ThumbImageSource

<!-- Upstream: RXT331. -->

## Cause

A MAUI `<Slider>` sets both `ThumbColor` and `ThumbImageSource`.

## Rule description

`ThumbImageSource` unconditionally overrides `ThumbColor` when rendering
the slider's thumb. Setting both is not a compile or runtime error, but
the `ThumbColor` value silently has no visible effect — the next developer
to read the markup will be confused about which one "wins."

Any combination of literal and bound values triggers the rule; presence of
both attributes is the signal, regardless of value.

```xaml
<StackLayout xmlns="http://schemas.microsoft.com/dotnet/2021/maui">
    <!-- non-compliant: ThumbColor is ignored -->
    <Slider ThumbColor="Red" ThumbImageSource="thumb.png" />
</StackLayout>
```

```xaml
<StackLayout xmlns="http://schemas.microsoft.com/dotnet/2021/maui">
    <!-- compliant: pick one -->
    <Slider ThumbImageSource="thumb.png" />
</StackLayout>
```

## How to fix violations

Remove whichever attribute is not needed. If the image is intentional,
delete `ThumbColor`; if the image was copy-pasted in, delete
`ThumbImageSource`.

## How to suppress violations

For a single element:

```xaml
<!-- xaml-lint disable once LX506 -->
<Slider ThumbColor="Red" ThumbImageSource="thumb.png" />
```

For a whole file or project:

```json
{ "rules": { "LX506": "off" } }
```
