# LX0105: Zero-sized RowDefinition / ColumnDefinition

<!-- xaml-lint-original; no upstream RXT equivalent. -->

**Applies to:** `Avalonia`  `MAUI`  `Uno`  `UWP`  `WinUI 3`  `WPF`

## Cause

A `<RowDefinition Height="…"/>` or `<ColumnDefinition Width="…"/>` carries a literal
size value that renders as zero pixels at runtime. The row or column silently
disappears — any children pinned to it via `Grid.Row` / `Grid.Column` collapse onto
adjacent rows / columns, often producing a layout the author did not intend.

The rule fires on three classes of literal value:

- Zero (`"0"`, `"0.0"`).
- Negative (`"-1"`, `"-0.5"`).
- Empty or whitespace-only (`""`, `"   "`).

```xaml
<!-- non-compliant: zero-height row disappears -->
<Grid>
    <Grid.RowDefinitions>
        <RowDefinition Height="0" />
        <RowDefinition Height="*" />
    </Grid.RowDefinitions>
    <Button Grid.Row="0" Content="invisible" />
    <Button Grid.Row="1" Content="OK" />
</Grid>
```

```xaml
<!-- non-compliant: empty Width parses as zero on every dialect -->
<Grid>
    <Grid.ColumnDefinitions>
        <ColumnDefinition Width="" />
    </Grid.ColumnDefinitions>
</Grid>
```

```xaml
<!-- compliant: Auto, *, and positive pixel values all render something -->
<Grid>
    <Grid.RowDefinitions>
        <RowDefinition Height="Auto" />
        <RowDefinition Height="2*" />
        <RowDefinition Height="32" />
    </Grid.RowDefinitions>
</Grid>
```

## Rule description

Only literal attribute-syntax values are checked. The parser is deliberately narrow:

- A value that ends with `*` is star-sized (`"*"`, `"2*"`, even `"0*"`) — the row /
  column participates in proportional sizing and is not a fixed zero. Not flagged.
- `"Auto"` (case-insensitive) is auto-sized. Not flagged.
- A markup-extension value (`"{Binding RowHeight}"`) is not a literal and cannot be
  reasoned about statically. Not flagged.
- Any other string that does not parse as an invariant-culture `double` is skipped.
- Among parseable literal numbers, only zero and negative values fire.

### What is out of scope

- **Element syntax.** `<RowDefinition><RowDefinition.Height>0</…></…>` is not inspected.
  The common spelling is the `Height=` attribute; v1.2 ships the common case and
  leaves the element form for a later minor.
- **`0*` (zero-weight star sizing).** `0*` is a valid GridLength that deliberately
  takes no slack — sometimes used for animated show / hide. Not flagged.
- **Positive fractional values under 1** (`"0.5"`). Still render something non-zero;
  not flagged.
- **Grid ancestry is not checked.** A `<RowDefinition Height="0"/>` outside a `<Grid>`
  is structurally wrong regardless; the rule flags it anyway.

## How to fix violations

- Replace the literal with `"Auto"` or `"*"` — almost always the intended sizing.
- If the row or column really should be hidden at times, bind the `Height` / `Width`
  to a view-model property (markup-extension values are never flagged) and drive it
  from view-model state.

## How to suppress violations

For a single element (the next `XElement` after the comment):

```xaml
<!-- xaml-lint disable once LX0105 -->
<RowDefinition Height="0" />
```

For a block of markup:

```xaml
<!-- xaml-lint disable LX0105 -->
<RowDefinition Height="0" />
<ColumnDefinition Width="" />
<!-- xaml-lint restore LX0105 -->
```

For a whole file or project:

```json
{ "rules": { "LX0105": "off" } }
```
